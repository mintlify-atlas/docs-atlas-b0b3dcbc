## Overview

Process large volumes of comments efficiently with SentimentAPI's bulk CSV upload feature. Upload a CSV file containing hundreds or thousands of comments and receive sentiment analysis results for all of them in seconds.

<Info>
Bulk processing is perfect for analyzing survey responses, product reviews, or social media comments at scale.
</Info>

### File Size Validation

The system validates file size before upload to prevent oversized files:

```typescript upload-csv/page.tsx
const MAX_SIZE_MB = 3 * 1024 * 1024;
if (selectedFile.size > MAX_SIZE_MB) {
  setMessage(`El archivo es demasiado pesado (${(selectedFile.size / (1024 * 1024)).toFixed(2)} MB). Por favor, sube uno menor a 3MB.`);
  setUploadStatus('error');
  return;
}
```

<Note>
Reference: `app/sentiment/upload-csv/page.tsx:138-143`
</Note>

<Warning>
Files larger than 3MB will be rejected. If you need to process larger datasets, split them into multiple files or contact support for enterprise options.
</Warning>

## Expected CSV Format

Your CSV file should contain comments in a structured format. Here's the expected structure:

```csv
texto
"Me encanta este producto"
"No funcionó como esperaba"
"Excelente servicio al cliente"
"La entrega fue muy lenta"
```

<Tip>
Ensure your CSV has a header row with the column name `texto` for the comment field.
</Tip>

### Multi-column Support

If your CSV contains additional columns, the API will extract the `texto` column:

```csv
id,texto,fecha
1,"Producto excelente",2026-03-10
2,"No lo recomiendo",2026-03-11
3,"Muy satisfecho con la compra",2026-03-12
```

## Upload Process

### 1. Select File

The upload interface provides a drag-and-drop area or file selector:

```typescript upload-csv/page.tsx
const handleFileChange = (event: React.ChangeEvent<HTMLInputElement>) => {
  if (event.target.files && event.target.files[0]) {
    setSelectedFile(event.target.files[0]);
    setUploadStatus(null);
    setMessage('');
  }
};
```

### 2. Submit for Analysis

Once selected, the file is sent to the API for processing:

```typescript upload-csv/page.tsx
const formData = new FormData();
formData.append('file', selectedFile);

const response = await fetch(`${process.env.NEXT_PUBLIC_API_URL}/sentiment/upload-csv`, {
  method: "POST",
  headers: { "Authorization": `Bearer ${token}` },
  body: formData,
});

const data = await response.json();
setResultados(data);
```

### 3. View Results

Results are displayed in a structured table showing:

- Original comment text
- Sentiment prediction (POSITIVO/NEGATIVO)
- Confidence probability
- Visual indicators and progress bars

## Response Structure

The bulk analysis returns an array of sentiment results:

```typescript
interface AnalisisResultado {
  texto: string;
  prevision: 'POSITIVO' | 'NEGATIVO';
  probabilidad: number;
}
```

## Error Handling

The system provides detailed error messages for common issues:

<AccordionGroup>
  <Accordion title="File Too Large (401/413)">
    ```typescript
    case 401:
      throw new Error("El archivo supera el límite de 3MB. Intenta con un archivo más pequeño.");
    case 413:
      throw new Error("El servidor indica que el archivo es demasiado grande.");
    ```
    
    **Solution**: Split your CSV into smaller files or reduce the number of comments.
  </Accordion>
  
  <Accordion title="Authentication Error (403)">
    ```typescript
    case 403:
      throw new Error("No tienes permisos para realizar esta acción.");
    ```
    
    **Solution**: Verify your API token is valid and has the correct permissions.
  </Accordion>
  
  <Accordion title="Server Error (500)">
    ```typescript
    case 500:
      throw new Error("Error interno del servidor. Inténtalo más tarde.");
    ```
    
    **Solution**: Retry the upload. If the problem persists, contact support.
  </Accordion>
  
  <Accordion title="Connection Error">
    ```typescript
    if (err.message === "Failed to fetch") {
      setMessage("No se pudo conectar con el servidor. Verifica tu conexión.");
    }
    ```
    
    **Solution**: Check your internet connection and verify the API endpoint is accessible.
  </Accordion>
</AccordionGroup>
